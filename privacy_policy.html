DiscPost Privacy Policy / プライバシーポリシー

Last updated: October 31, 2025

Effective Area / 適用範囲

This Privacy Policy explains how DiscPost("DiscPost" or "we" or "us"),the Chrome Extension, collects, uses, and discloses your information. This Policy applies only to the DiscPost,the Chrome Extension, and not to any websites or services accessed through it.

本プライバシーポリシーは、Chrome拡張機能であるDiscPost（以下「DiscPost」）が ユーザーの情報をどのように収集、使用、開示するかを説明します。 本ポリシーは、DiscPost(Chrome拡張機能)にのみ適用され、 拡張機能がアクセスする他のサービスには適用されません。

Policy Changes / 変更範囲

The provisions of this Policy may be changed at any time and will indicate when changes have been made by revising the date at the top of this Policy. We encourage you to review this Policy whenever you access the Service to ensure you understand our information collection, use, and disclosure practices.

本ポリシーは変更される可能性があります。 その変更は本ポリシーの冒頭に記載された日付を改定することにより通知されます。 本サービスにアクセスする際は、DiscPostの情報収集、使用、開示の慣行を理解するために、本ポリシーを随時確認されることを推奨します。

Handling of Personal Information / 個人情報の取り扱い

DiscPost does not handle any PII,Personally Identifiable Information, such as names, ages, addresses, email addresses or others.

氏名、年齢、住所、メールアドレス等のいかなる個人情報も取り扱いません。

Children's Privacy / 児童のプライバシー

DiscPost is not intended for use by children under 13 years of age. We do not knowingly collect any personal information from children.

DiscPostは13歳未満の児童を対象としていません。 また、児童から個人情報を意図的に収集することはありません。

Scope and Methods of Information Collection / 収集する情報の範囲と取得方法

DiscPost strives to protect user privacy by minimizing the information collected. The information collected is as below:

Highly Sensitive Information

The current tab's URL is considered highly sensitive information and is handled with the utmost care. It is collected only upon the user's explicit action (a clear indication of intent) and strict conditions:

  • The current tab's URL is never automatically collected without the user's explicit action
  • The explicit consent message is displayed to users
  • It is only temporarily acquired in memory if the user provides explicit consent beforehand.
  • This consent process is performed each time the extension's popup is invoked.
  • The current tab's URL is never stored in storage (permanently and temporarily)

Less Sensitive Information

This information is collected via the user's explicit input:

  • WEBHOOK URL
  • WEBHOOK Name
  • Prefix
  • Draft Message

DiscPostは、収集する情報を最小限に抑え、ユーザーのプライバシーを保護に努めます。 収集する情報は以下の通りです。

機密性の高い情報

現在のアクティブなタブのURLは機密情報と見なされ、最大限の注意を払って取り扱います。以下の厳格な条件でのみ収集されます。

  • 現在のアクティブなタブのURLは自動的に収集されることは一切ありません。
  • ユーザーには同意を求めるメッセージが表示されます
  • 事前にユーザーから同意を得た場合にのみ、一時的に取得（メモリ上に保持）されます。
  • この同意プロセスは拡張機能のポップアップを呼び出す都度、行われます。

機密性の低い情報

この情報は、ユーザーの明示的な入力により収集されます。

  • WEBHOOKのURL
  • WEBHOOKの名称
  • プレフィックス
  • 下書きメッセージ

Purpose of Use of Collected Information / 取得した情報の利用目的

DiscPost uses the collected information solely to achieve the purpose of the extension:

Information Used for Message Transmission

  • WEBHOOK URL: Used to send the message created by the user to the specified Discord channel (the core function of the extension).
  • Active Tab's URL: Used to embed the current URL into the message when the user submits it, serving as a message creation aid.

Information Used for Message Creation

  • WEBHOOK Name: Used by the user to identify the WEBHOOK URL.
  • Prefix: Used by the user to create messages that are easily identifiable by specific keywords (a message creation aid).
  • Draft Message: Used by the user to create the message content.

DiscPostは、収集した情報を拡張機能の目的を達するためにのみ利用します。

メッセージを送信するために利用する情報

  • WEBHOOKのURL: ユーザーが作成したメッセージをDiscordの指定のチャンネルに送信するために利用します。（拡張機能の中核的な機能です。）
  • アクティブなタブのURL: ユーザーがメッセージを送信する際、現在のURLをメッセージに組み込むために利用します。（ユーザーのメッセージ作成を補助する機能です。）

メッセージを作成するために利用する情報

  • WEBHOOKの名称:ユーザーがWEBHOOK URLを識別するために利用します。
  • プレフィックス:ユーザーが特定のキーワードで識別しやすいメッセージを作成するために利用します。（ユーザーのメッセージ作成を補助する機能です。）
  • 下書きメッセージ:ユーザーがメッセージを作成するために利用します。

Storage and Disposal of Collected Information

DiscPost stores the following collected information in local storage using chrome.storage.local to achieve the purpose of the extension. Users can dispose of the stored information at any time.

  • WEBHOOK URL, Name, and Prefix: Persistently stored as settings information until the user explicitly deletes them via the Delete button on the settings screen.
  • Draft Message: Stored temporarily until the user sends or discards the data. The user can clear it at any time using the Delete button on the posting screen.

DiscPost does not store the following collected information:

  • Active Tab's URL: It is never stored, either persistently or temporarily.

DiscPostは拡張機能の目的を達するために、収集した以下の情報をchrome.storage.localを利用して、ローカルストレージで保存します。ユーザーは保管された情報をいつでも破棄することができます。

  • WEBHOOKのURL、名称、プレフィックス: 設定情報としてユーザーが設定画面のデリートボタン（消去）でいつでも消去できます。
  • 下書きメッセージ: 入力後、ユーザーがデータ送信もしくは破棄をするまで一時的に保存されます。ユーザーは投稿画面のデリートボタン（消去）でいつでも消去できます。

DiscPostは、収集した以下の情報の保存を一切行いません。

  • アクティブなタブのURL: 永続的・一時的にかかわらず、保存されることは一切ありません。

Disclosure of Collected Information to Third Parties / 取得した情報の第三者提供

DiscPost does not disclose information to any third-party advertisers, analytics providers, or other entities. Information is stored only in the user's local storage, and the developers have no means of accessing this information.

  • Disclosure to Discord: The Webhook URL and message content (including the current tab URL if consented to) are transmitted only to the Discord API according to the user's explicit instruction.

DiscPostは、いかなる第三者の広告主、分析プロバイダー、その他の団体にも情報を開示することはありません。 また、情報の保存はユーザーのローカルストレージにのみ行っており、開発者はその情報にアクセスする手段を持ちません。

  • Discordへの提供: Webhook URLとメッセージ内容（同意された場合は現在のタブのURLを含む）は、お客様の明示的な指示に従い、Discord APIにのみ送信されます。

Use of External Services / 外部サービスの利用

DiscPost utilizes the following external service. Please refer to the privacy policy of the respective service for information handling:

  • Discord: Sent to Discord using the WEBHOOK URL registered by the user as the endpoint.

DiscPostは、以下の外部サービスを利用します。外部サービスの情報の取り扱いについては各サービスのプライバシーポリシーを参照してください。

  • Discord: ユーザーが登録したWEBHOOK URLをエンドポイントとして、Discordに送信されます。

Information Security / 安全管理措置

DiscPost implements reasonable technical and measures to protect the information handled within the extension from unauthorized access, alteration, disclosure, or destruction.

  • Note: Since all information is stored locally, protection also depends on the user's device security.

拡張機能内で取り扱う情報を不正アクセス、改ざん、開示、破壊等から保護するため、 合理的な技術的および措置を講じます。

  • 注意: すべての情報はローカルストレージに保管されます。 よって、セキュリティはユーザーのデバイスに依存します。
